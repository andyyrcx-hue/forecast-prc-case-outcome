# forecast-prc-case-outcome

面向中国大陆法院诉讼研究的 Codex Skill：检索事实相似类案、权威裁判和承办法官公开裁判样本，分别评估事实相似度、权威参照级和法官样本强度，形成条件化裁判预测与诉讼策略。

> 本项目不是“算胜诉率”的工具，也不保证判决结果。它用于辅助律师更快看见裁判趋势、识别证据风险并组织诉讼论证，不能替代律师对事实、证据、法律效力和诉讼策略的最终判断。

> [!IMPORTANT]
> **使用前必须另行安装并连接元典案例 MCP。**
>
> 本仓库只包含 Codex Skill，**不包含元典 MCP、元典账号、API Key 或访问令牌**。仅安装本 Skill，无法完成真实的案例检索、裁判文书全文核验和承办法官公开裁判样本分析。
>
> 请先自行取得元典案例服务的合法访问权限，再按照元典提供的最新说明完成 MCP 安装和配置。接口名称和配置方式可能随版本变化，请以用户实际取得的官方说明和工具 schema 为准。

## 能做什么

- 将案情拆成法律关系、争议焦点、决定性事实、证据结构和目标判项；
- 通过元典案例 MCP 检索权威案例、事实相似案例、反面案例和候选法官样本；
- 分开计算事实相似度 `S 0—100`、权威参照级 `A1—A6` 和法官样本强度 `J0—J4`；
- 有利案例强调与胜负开关有关的相同点，不利案例识别可能改变判项的真实差异；
- 比较上级法院裁判规则与承办法官公开裁判样本是否一致；
- 输出有利、基准、不利三种情景，并提出补证、庭审准备、代理论证结构，以及是否需要考虑和解的内部风险提示；
- 每次完成分析后，同时生成《类案检索与裁判预测报告》的 Markdown 主编辑版和 DOCX 正式阅读版。

## 适用范围

仅适用于中国大陆法院公开裁判文书研究。暂不适用于：

- 仲裁裁决；
- 其他国家或地区的司法预测；
- 未经授权的客户材料外部传输；
- 根据搜索摘要、模型记忆或单个法官样本作确定性结论。

## 能力边界

本 Skill 的默认成果是内部类案研究与裁判预测报告，不是完整代理词、起诉状、答辩状、上诉状或其他可直接提交法院的诉讼文书。

本 Skill 可以为后续文书起草提供经核验的类案、裁判规则、证据对应关系和代理论证结构。需要起草具体诉讼文书时，应由用户另行明确提出，并结合案件材料、程序阶段、庭审情况和法院要求进行专项处理。

本 Skill 只能根据预测结果提示是否需要考虑和解，并列出需要用户进一步确认的谈判边界；不默认计算具体和解金额，也不生成完整谈判方案或和解协议。具体和解策略还需要结合诉讼成本、执行可能性、当事人目标、授权范围、履行能力和谈判信息另行分析。

## 安装

### 第一步：安装并连接元典案例 MCP

先按照元典提供的最新说明，在 Codex 中安装、连接和配置元典案例 MCP。本项目不代替元典提供 MCP 安装包、访问权限或配置凭据。

连接后，请先在 Codex 中测试：

```text
请检查当前是否可以调用元典案例检索工具，只报告工具是否可用，不要发送任何案件材料。
```

只有 Codex 能够识别并实际调用元典案例检索工具，才能继续进行真实类案分析。仅看到配置文件或工具名称，不等于连接已经可用。

### 第二步：安装本 Skill

将本仓库中的 Skill 目录复制到 Codex Skills 目录：

```bash
cp -R skills/forecast-prc-case-outcome ~/.codex/skills/
```

随后新建一个 Codex 任务，或重新加载 Skills。

> 如果你的 `CODEX_HOME` 不在默认位置，请将目录复制到 `$CODEX_HOME/skills/`。

### 第三步：验证 Skill 是否触发

新建一个 Codex 任务并输入：

```text
使用 $forecast-prc-case-outcome，先检查元典案例 MCP 是否可用，不要发送任何真实案情。
```

正常情况下，Codex 应同时识别本 Skill 和元典案例检索工具。如果元典 MCP 未安装、未连接或调用失败，Skill 应明确标注 `[连接器未核验]`，停止输出具体案例、案号和法官倾向，不得使用模型记忆补造检索结果。

## 安装顺序一览

1. 取得元典案例服务的合法访问权限；
2. 另行安装并连接元典案例 MCP；
3. 验证 Codex 能够实际调用元典案例检索工具；
4. 安装 `forecast-prc-case-outcome` Skill；
5. 新建 Codex 任务，验证 Skill 触发；
6. 对案情脱敏后，再开始真实类案检索。

## 触发方式

最稳妥的显式触发：

```text
使用 $forecast-prc-case-outcome 分析这个案子。
```

也可以直接说：

- “查一下类似案例的判决倾向。”
- “类案预测：这个案子可能怎么判？”
- “查承办法官同类案件的裁判倾向。”
- “按最高法、高院、中院和基层法院分级检索类案。”

推荐输入模板：

```text
类案预测：我是原告/被告，案件由××法院审理，核心争议是……，
我方目标判项是……，现有关键证据是……。
请检索相似案例，按法院权威层级整理，分析承办法官公开裁判样本，
并预测本案可能怎么判。
```

## 输出成果

默认同时生成：

- `类案检索与裁判预测报告.md`：主编辑、持续更新版本；
- `类案检索与裁判预测报告.docx`：正式阅读版本；
- `检索记录与关键词.md`：脱敏检索式、数据源、检索日期、全文核验及排除记录。

报告包括：结论摘要、本案比较基准、权威轨、事实相似案例排序、承办法官轨、有利案例运用、不利案例区分、上下级观点冲突、三情景预测、行动建议及检索局限。其中，行动建议可以包含代理论证结构，但不等同于生成完整代理词或其他可直接提交法院的诉讼文书。

## 三套评分为什么分开

- `S` 回答“是不是类案”；
- `A` 回答“这个观点有多强的参照价值”；
- `J` 回答“能否说明承办法官存在相对稳定的公开裁判观察”。

三者不合成一个不透明的“胜诉指数”。有利结果不会提高相似度，法院级别也不会提高事实相似度。

## 隐私与安全

- 调用外部检索前，先删除或替换姓名、企业名、案号、合同编号、精确金额、联系方式、流水、证据原文和商业秘密；
- 法官姓名只有在用户明确要求检索该法官公开裁判并授权发送时，才作为外部查询条件；
- 不在仓库、示例、日志或检索记录中保存 API Key、访问令牌、MCP 配置秘密或真实客户信息；
- 无法取得全文时标注 `[摘要命中—待全文核验]`，不得把摘要包装成已核验裁判观点。
