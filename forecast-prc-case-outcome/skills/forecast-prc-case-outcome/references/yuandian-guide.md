# 元典检索指南

> 本项目与元典不存在隶属、授权、合作或背书关系。接口名称和参数可能随 MCP 版本变化，以用户实际取得的 schema 和官方说明为准。

## 1. 连接探测

先实际调用轻量级案例检索能力。未暴露或调用失败时，标记 `[连接器未核验]` 并停止输出具体案例和法官倾向。配置文件存在不等于工具可用。

优先识别等价的案例检索和全文获取 capability。部分环境中可能出现以下底层名称：

| 目的 | 可能的工具名称 | 常见参数 |
| --- | --- | --- |
| 权威案例 | `yuandian_rh_qwal_search` | `qw`, `ay`, `ajlb`, `ja_start`, `ja_end`, `top_k` |
| 普通案例 | `yuandian_rh_ptal_search` | `qw`, `fxgc`, `ajlb`, `wszl`, `xzqh_p`, `yyft`, `top_k` |
| 事实语义检索 | `yuandian_case_vector_search` | `query`, `wenshu_type`, `dianxing`, `cj`, `return_num` |
| 案例全文 | `yuandian_rh_case_details` | `type` + `ah` 或 `id` |

只传当前工具 schema 支持的字段，不猜测参数。接口 schema 与本表冲突时，以实际 schema 为准。

## 2. 固定检索顺序

1. 用核心争点检索权威案例；
2. 用案由、争点、适用规范检索普通案例；
3. 用脱敏自然语言事实做语义补充；
4. 用反面事实组合主动检索不利案例；
5. 对候选核心案例逐一拉取全文；
6. 经授权后，用法官姓名结合案由/争点检索，并在全文末尾核验署名。

每个争点至少进行一组同义词或旧法表述扩展。命中不足不代表不存在相反案例。

## 3. 法官轨检索

若接口没有结构化法官字段，使用法官姓名作为全文关键词分批检索，随后逐案获取全文并核对署名。不得仅凭标题、摘要或同法院信息推断承办法官。

核验：姓名、任职法院与时期、审判领域、署名角色、独任或合议庭、是否处理目标争点、是否存在重名或调任。只找到同法院样本时，标题写“本院裁判倾向”，不得写“该法官倾向”。

## 4. 来源标签

- `[元典全文已核验]`：本次任务已取得全文并核对关键字段；
- `[元典摘要命中—待全文核验]`：只有搜索摘要，不得作为核心引用；
- `[身份待核验]`：法官同名或署名角色仍有疑问；
- `[待核验生效/后续程序]`：未确认是否被改判、撤销或进入再审；
- `[待检索]`：工具不可用或尚未执行。

禁止使用模型记忆案例补足样本数量。

## 5. 检索记录

记录日期、数据库、脱敏查询、参数、命中总数、拉取全文数、纳入/排除理由、法官身份核验方法和数据库覆盖局限。不要记录 API Key、访问令牌或 MCP 配置秘密。
